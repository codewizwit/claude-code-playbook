# How I Work

I'm an educator creating [CONTENT, e.g. curriculum, lesson plans, workshops, training materials] for [AUDIENCE, e.g. students, professionals, bootcamp learners, corporate teams].

## Workflow

### Learner First

- Start with what the learner already knows. Build from there.
- Every lesson needs a clear outcome. "After this, you'll be able to..."
- Default to plan mode for new curriculum structures.

### Active Learning

- Prefer hands-on exercises over lectures. People learn by doing.
- Include checkpoints where learners verify their understanding.
- Provide worked examples before asking learners to solve problems independently.

### Inclusive Design

- Write for varied experience levels. Offer stretch goals, not gatekeeping.
- Use plain language. Jargon is a barrier unless the jargon itself is the lesson.
- Provide multiple entry points. Not everyone learns the same way.

## Content Types

- [PRIMARY]: [e.g. course modules, workshops, tutorials, assessments]
- [TOOLS]: [e.g. Markdown, slides, interactive notebooks, video scripts]
- [AUDIENCE]: [e.g. beginners, intermediate practitioners, executives]

## Standards

- Learning objectives use measurable verbs (explain, implement, evaluate). Not "understand."
- Every exercise has expected output so learners can self-check.
- Code examples work. Copy, paste, run.
- Assessment questions test application, not memorization.
