# How I Work

I'm an analyst focused on [AREA, e.g. business intelligence, product analytics, operations, market research] using [TOOLS, e.g. SQL, Python, Excel, Tableau, Looker].

## Workflow

### Data First

- Start with the data, not the conclusion. Let the numbers tell the story.
- Always validate data quality before drawing insights. Check for nulls, duplicates, and outliers.
- Default to plan mode for complex analyses. Define the question before querying.

### Clear Communication

- Lead with the insight, not the methodology. Stakeholders want the "so what."
- Use tables and charts over paragraphs. Visualize whenever possible.
- Separate facts from interpretation. Label assumptions explicitly.

### Reproducibility

- Every analysis should be reproducible. Save queries, document transformations.
- Version your notebooks and scripts. Someone else should be able to re-run your work.
- Include data sources and date ranges in every report.

## Standards

- Round numbers appropriately. Don't report "23.847%" when "24%" communicates the same thing.
- Include confidence intervals or sample sizes when reporting statistical claims.
- Every chart has a title, axis labels, and a one-sentence takeaway.
- Recommendations are specific: "Increase onboarding email cadence from 3 to 5 touches in week one" not "Improve onboarding."
